reference
I need to remember something so I can use it later
Use a variable. Useful variable types include: String, integer, float, and boolean. 

  String a = "To be or not to be";
  int b = 100;
  float c = 3.5;
  boolean c = false;
I need to make a decision
Use an if statement, also known as a conditional statement 

  if (x < 100) {
    
  } else {
    
  }
or 

  if (name=="Ben") {
    
  } else {
    
  }
I need to draw a shape.
Use a shape, such as a point, line, rect, or ellipse. 

  point(50,50);
  line(0,0,10,10);
  rect(10,10,100,100);
  ellipse(50,50,10,10);
I need to save an action so I can do it later
Use a function. See reference for void 

  void changeBG() {
   
    background(0);
  
  }
Later, you can invoke it: 

  changeBG()
I need my action to happen a little bit differently each time
Use a function with an argument. 

  void changeBG(color) {

    background(color);

  }
Later, you can invoke it with different arguments. 

  changeBG(255);
or 

  changeBG(0);
I need to change the color or style of my shape
Set the fill or stroke colors. 

  stroke(255,0,0);
  fill(100,100,0);
more advanced
I need to remember a lot of things that are related somehow
Use an array 

  Array groceries = [ "milk", "eggs", "bread" ];
This invocation will return "milk": 

  groceries[0];
I need to do something several times
Use a loop 

  for (var i=0;i<100;i++) {
    //the code in here will happen 100 times
  }
I need to make something on my site interactive
Use a built-in interaction function like mousePressed(). This function will be called automatically when the mouse is pressed. 

  void mousePressed() {

    point(mouseX, mouseY);

  }
I want to know where the mouse is.
Use an built-in variables mouseX and mouseY 

  void draw() {

    line(pmouseX, pmouseY, mouseX, mouseY);

  }

whether to turn the toggle on or toggle off
I need to draw a shape
to visually represent the change in the toggle's state
boolean on = false;
void mousePressed() {
if (on==false) {
fill(0,255,0)
rect(10,10,100,100)
on = true
} else {
fill(200,200,200)
rect(10,10,100,100)
on = false
}
}
Remember, there are a dozen ways to code any given idea. It is usually easier to understand your own code than to read someone else's. Steal the essentials and go be creative!
